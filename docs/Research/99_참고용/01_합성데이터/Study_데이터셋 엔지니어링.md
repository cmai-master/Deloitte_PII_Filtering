**데이터 큐레이션**

1. 고품질 데이터란?  
   1. 여섯가지 특성(관련성, 작업 요구사항 부합, 일관성, 올바른 형식, 충분한 고유성, 규정 준수)을 가진 데이터  
2. 데이터가 적다면 더 고급 모델에 PEFT(Parameter-Efficient Fine-Tuning) 방법을 사용하는 것이 좋다  
3. 대규모 데이터셋 구축 투자 전, 먼저 잘 만들어진 소규모 데이터셋을 시작해서 파인튜닝이 모델을 개선할 수 있는지 확인해 보는 것이 좋다

**데이터 증강 및 합성**

4. 라이브러리: 테스트용 가짜 데이터 생성 (이름, 주소 등)  
   1. [https://github.com/joke2k/faker](https://github.com/joke2k/faker)  
   2. [https://chancejs.com/](https://chancejs.com/)  
5. 규칙 기반  
   1. 템플릿을 만들고 Faker 같은 난수 생성기로 각 필드 채우기  
6. 섭동(Perturbation)  
   1. 기존 데이터에 노이즈를 넣어서 새로운 데이터를 생성  
      : 모델 성능을 향상시키고 공격에 더 견고하게 만들 수 있음  
   2. BERT를 학습시킬 때 연구자들은 토큰의 1.5%를 무작위 단어로 교체했음  
7. 지시 데이터 합성  
   1. 지시 생성: 포함하고 싶은 주제, 키워드, 지시 유형 목록으로 시작. 그다음 목록의 항목마다 일정 수의 지시를 만든다. 템플릿 셋트로 시작해서 템플릿당 일정 수의 예시를 만드는 방법도 존재.  
   2. 응답 생성: 지시 하나당 하나 또는 여러개의 응답을 만들 수 있음  
8. 데이터 검증  
   1. 생성된 데이터의 품질을 보장하기 위해 엄격한 정확성 분석과 오류 수정 과정을 거침 (AI 활용)  
   2. AI 검증기  
      1. 각 예시에 1점부터 5점, 좋은 또는 나쁨으로 분류  
      2. 파운데이션 모델에 품질 요구사항을 설명하고 데이터 예시가 이런 요구사항을 만족하는지 판단  
9. AI 합성 데이터의 한계  
   1. 품질 관리: Garbage in Garbage out  
   2. 피사적 모방: 사실적 정확성과 학습 데이터 범위를 벗어난 과제에 대한 일반화에서는 어려움을 겪을 수 있음  
   3. 모델 성능 저하 가능성: 모델 붕괴. 시간이 지날 수록 더 일반적인 사건을 출력하면서 희귀한 사건을 잊게 됨. 모델이 이전 모델 출력을 포함한 데이터셋으로 학습되면 모델의 기존 편향이 더 커질 수 있음.  
   4. 불분명한 데이터 계보: 학습 데이터의 영향을 받고 때로는 사용자도 모르게 그 내용을 그대로 출력함.
