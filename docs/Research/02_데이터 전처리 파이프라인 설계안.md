# **0\. 문서 개요**

본 문서는 LG U+ Server-i 플랫폼의 PII 오탐·정탐 분류 모델 구축을 위한 데이터 전처리 파이프라인의 상세 설계서입니다. 3/19 회의 결정사항과 미결 리서치 항목을 함께 관리합니다.

| 📌 설계 범위 및 전제 • 대상 데이터: 마스킹(Asterisk)된 PII 패턴 텍스트 \+ 앞뒤 \~10자 컨텍스트 • 분류 목표: TP(정탐) 1종 \+ FP(오탐) 7종, 총 8개 클래스 • 학습 데이터: 합성 데이터(Train 80% / Val 20%) → 실데이터(Test holdout) |
| :---- |

## **1\. 데이터 전처리 파이프라인 (Preprocessing Pipeline)**

Raw 텍스트 및 Tabular 데이터를 모델이 처리할 수 있는 형태로 변환합니다. 전처리 규칙 변경이 모델 성능에 직결되므로 모든 변경은 버전 관리됩니다.

| PreprocessingPipeline ├── 1.1  TextNormalizer      \# 대소문자·공백·패턴 정규화 ├── 1.2  MaskTokenEncoder    \# \*\*\*\* → \[MASK\_N\] 변환 ├── 1.3  Tokenizer           \# KLUE-RoBERTa / DeBERTa 등 (모델과 쌍) ├── 1.3  SequencePadder      \# max\_length 패딩·트런케이션 ├── 1.3  TabularScaler       \# Min-Max Scaling (검토 중) ├── 1.3  LabelEncoder        \# 클래스 → 인덱스 (0\~7) ├── 1.5  EdgeCaseHandler     \# 결측·OOV 방어 로직 └── 1.6  DataSplitter        \# Train 80% / Val 20% / Test(실데이터) |
| :---- |

## **1.1  텍스트 정규화 (TextNormalizer)**

| 📋  \[3/19 회의 결정사항\] • 전처리 순서(대소문자 변환 등)가 모델 성능에 큰 영향을 미치므로, 실제 데이터를 활용한 시뮬레이션 필수. • 어떤 형태의 데이터가 들어와도 원하는 폼으로 변환할 수 있도록 파이프라인을 정교하게 다듬어야 함. |
| :---- |

| 처리 항목 | 변환 규칙 | 예시 |
| :---- | :---- | :---- |
| **대소문자 통일** | 모든 텍스트를 소문자로 변환 | \*\*\*\*@BPS.LGUPLUS.CO.KR → \*\*\*\*@bps.lguplus.co.kr |
| **공백 처리** | 양 끝 공백 제거, 연속 중복 공백 단일 공백으로 축소 | '  hello  world  ' → 'hello world' |
| **URL·경로 보존** | / 경로 구조는 분류 단서이므로 변경 없이 보존 | /test/, /dev/, /sample/ 패턴 유지 |

## **1.2  마스킹 토큰 인코딩 (MaskTokenEncoder)**

'\*\*\*\*' 패턴을 마스킹 자리수가 명시된 전용 특수 토큰으로 변환합니다. 마스킹 길이 자체가 클래스 구분의 핵심 단서입니다.

| 📌 마스킹 길이가 클래스 구분 단서인 이유 • FP-3 (Timestamp): xpiryDate=170603\*\*\*\*\* → \[MASK\_5\] — 날짜·시간 자리수 패턴 • FP-5 (Bytes): 141022\*\*\*\*\*\* (1.2 7/0) → \[MASK\_6\] — 긴 숫자열 바이트 패턴 • FP-4 (내부 도메인): \*\*\*\*@bdp.lguplus.co.kr → \[MASK\_4\]@bdp.lguplus.co.kr • 같은 이메일 패턴이라도 마스킹 길이 \+ 도메인 조합으로 FP-2/FP-4/FP-6 구분 가능 |
| :---- |

| \# 변환 예시 입력:  '\*\*\*\*@bdp.lguplus.co.kr' 출력:  '\[MASK\_4\]@bdp.lguplus.co.kr' 입력:  'xpiryDate=170603\*\*\*\*\*' 출력:  'xpiryDate=170603\[MASK\_5\]' |
| :---- |

| ⚠  미결 ▸ Truncation 방향(pre/post) 결정 필요: 텍스트가 max\_length 초과 시 앞을 자를지(pre) 뒤를 자를지(post) 결정 → post (뒷부분 자르기): 앞쪽에 핵심 패턴이 있을 때 유리 (예: FP-1 코드 패턴) → pre (앞부분 자르기): 뒤쪽 도메인·확장자가 분류 단서일 때 유리 (예: FP-4 @lguplus) → 결정 기준: 실데이터에서 핵심 분류 단서가 문자열의 앞/뒤 중 어디에 주로 위치하는지 먼저 분석 필요 |
| :---- |

## **1.3  토크나이저 · 패딩 · 인코딩**

**토크나이저 (Tokenizer)**

사전학습 모델과 토크나이저는 반드시 쌍으로 사용해야 합니다. Text·Tabular 인코더 모델이 다를 경우 각각의 전용 토크나이저가 필요합니다.

1. Text data Tokenizer: mDeBERTa-v3-base  
2. Tabular data Tokenizer: FT-Transformer

**시퀀스 패딩 (SequencePadder)**

GPU 병렬 연산을 위해 배치 내 모든 텍스트 길이를 동일하게 맞추는 필수 단계입니다.

| ⚠  미결 ▸ 최적 Max Length 산출: 전체 합성 데이터를 토큰화 후 길이 분포 EDA → 95\~99% 포함 길이를 max\_length로 설정 → 현재 임시 기준: 64 (실험으로 검증 필요 — 32 / 64 / 128 비교) → Trade-off: max\_length가 너무 크면 \[PAD\] 토큰 증가 → 메모리·연산 낭비 / 너무 작으면 정보 손실 |
| :---- |

**정형 데이터 스케일링 및 레이블 인코딩**

| ⚠  미결 ▸ Tabular 26개 피처 중 실제로 Min-Max Scaling이 필요한 연속형 피처 확인 필요 (예: 파일 크기, 검출 위치 등) |
| :---- |

| 처리 | 방법 | 비고 |
| :---- | :---- | :---- |
| **Min-Max Scaling** | 연속형 피처를 0\~1 범위로 정규화 | 적용 대상 피처 선별 후 결정 |
| **LabelEncoder** | 8개 클래스 → 정수 인덱스 (0\~7) | TP:0, FP\_숫자:1, FP\_더미:2, FP\_TS:3, FP\_도메인:4, FP\_bytes:5, FP\_CR:6, FP\_맥락:7 |

## **1.5  예외 및 극한 상황 처리 (Edge Cases & Fallbacks)**

운영 환경에서 예상치 못한 입력이 들어올 때의 방어 로직입니다.

| ⚠  미결 ▸ \[결측 텍스트\] Text 필드가 비어있거나 NaN·길이 0인 문자열 유입 시 처리 방법 결정 필요 → 옵션 A: \[PAD\] 토큰만으로 채워 모델에 통과 (모델이 빈 입력 처리하도록 학습) → 옵션 B: 전처리 단에서 에러 레코드로 분류 후 별도 버킷 격리 (권장 — 데이터 품질 관리) ▸ \[OOV 텍스트\] 토크나이저가 모르는 특수기호·외국어 유입 시 \[UNK\] 토큰 처리 허용치 설정 필요 ▸ \[OOV 정형\] Test·실전 데이터에 처음 등장하는 도메인/코드 값 → \<기타\> 또는 \-1 인덱스 매핑 로직 설계 필요 |
| :---- |

## **1.6  데이터 분할 및 버전 관리 (Data Splitting & Versioning)**

| 분할 | 데이터 소스 | 비율 | 용도 |
| :---- | :---- | :---- | :---- |
| **Train** | 합성 데이터 | 80% | 모델 가중치 업데이트 |
| **Validation** | 합성 데이터 | 20% | 학습 중 과적합 모니터링 |
| **Test (Holdout)** | 실제 현업 데이터 | 최대한 배정 | 최종 실전 성능 측정 (TSTR) |

| 📌 버전 관리 디렉토리 구조 • data/raw/              — 합성 원본 (변경 금지, 스냅샷 고정) • data/processed/v1.0/   — 전처리 완료 데이터 (규칙 변경 시 버전 올림) • data/splits/           — train/val/test 인덱스 고정 저장 (재현성 확보) • ⚠ 전처리 규칙 변경 \= 모델 성능 변경 → 반드시 버전 구분 후 실험 |
| :---- |
